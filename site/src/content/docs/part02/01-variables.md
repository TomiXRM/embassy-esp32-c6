## このページでできるようになること

- `let` を使って変数を宣言できる
- 型推論（コンパイラが型を自動で決めてくれる仕組み）を説明できる
- 自分の書いたコードを2通りの方法（開発ボード / Rust Playground）で動かせる

## 先に結論

Rustで値に名前をつけるには `let` を使います。`let width = 40;` のように書くと、以降 `width` という名前で 40 を使えます。C++と違って型を書かなくてもよいのは、コンパイラが右辺を見て型を自動で決めてくれるからです。これを**型推論**と呼びます。型を省略しても「型がない」わけではなく、コンパイル時には必ず1つの型に決まっています。

## コードを動かす2つの方法

第2部では、短いRustコードをたくさん動かして文法を覚えます。動かし方は2通りあります。どちらでも学べるので、好きなほうを使ってください。

**方法① 開発ボードで動かす（組み込み流）**

第1部で作ったblinkyプロジェクトの `src/main.rs` を開き、`info!("Lチカを開始します");` の行の直後（`loop {` の手前）に試したいコードを貼りつけます。そして第1部と同じように書き込みます。

```bash
cargo run
```

書き込みが終わると自動でログ画面（モニタ）になり、`log::info!` で書いた内容が表示されます。この部で扱うコードはすべて、開発ボードのRust環境（`no_std`という標準ライブラリなしの環境。第5部で説明します）でそのまま動く文法だけを使います。

**方法② Rust Playgroundで動かす（パソコンだけで手軽に）**

ブラウザで [Rust Playground](https://play.rust-lang.org/) を開くと、インストールなしでRustコードを実行できます。次の形の `main` の中に貼りつけて、Runボタンを押します。

```rust
fn main() {
    // ここに貼る
}
```

1つだけ読み替えが必要です。このページ以降のコードにある `log::info!(...)` を `println!(...)` に書き換えてください。`log::info!` は開発ボードでシリアル（USBケーブル越しの文字通信）にログを出すための書き方、`println!` はパソコンの画面に文字を出すための書き方で、カッコの中の書式はまったく同じです。

## 身近なたとえ

変数は「名前を書いたラベル付きの箱」です。`let width = 40;` は、「width」というラベルの箱を用意して 40 を入れる、というイメージです。あとから `width` と書くたびに、箱の中身の 40 が取り出されます。

ただし実際の変数は、マイコンのRAM（またはレジスタ）上に確保された記憶場所で、箱の大きさ（何バイト使うか）は型によってコンパイル時に決まっています。「どんな大きさのものでも入る魔法の箱」ではない、という点が本物との違いです。

## 仕組み

`let` の一行は、次の部品でできています。

```rust
let width: u32 = 40;
```

- `let` — 「これから変数を宣言します」という合図
- `width` — 変数の名前。英小文字とアンダースコア（`_`）でつなぐのがRustの習慣です（例: `wait_ms`）
- `: u32` — 型の指定（**型注釈**）。「32ビットの符号なし整数」という意味です（型の種類は[3. 数値型](/embassy-esp32-c6/part02/03-numbers/)で学びます）
- `= 40` — 箱に入れる最初の値
- `;` — 文の終わり

そして多くの場合、型注釈は省略できます。

```rust
let width = 40; // コンパイラが「整数だな」と推論する
```

コンパイラは右辺の `40` や、その変数があとでどう使われるかを見て、型を1つに決めます。これが型推論です。「型を書かなくてもいい」のであって「型がない」のではありません。推論できないときや、人間が読んで分かりやすくしたいときは型注釈を書きます。

## Arduinoではどう書くか

Arduino（C++）では型を自分で書くのが基本でした。

```cpp
int width = 40;        // C++: 型を先頭に書く
```

Rustでは `let` で始めて、型はコンパイラに任せられます。

```rust
let width = 40;        // Rust: 型はコンパイラが推論する
```

なお、新しめのC++にも `auto` という型推論がありますから、「C++にはできない」わけではありません。違いは、Rustでは型を省く書き方が最初から標準で、それでも型のチェックはC++と同じくらい厳しく行われる、という点です。

## RustとEmbassyではどう書くか

長方形の面積を計算して表示する例です。blinkyプロジェクトに貼るコードは、この部ではこのような「断片」で示します（これは抜粋です。貼りつけ先の完全なコードは examples/01-blinky を見てください）。

```rust
let width = 40;
let height = 25;
let area = width * height;
log::info!("面積は {} です", area);

let count: u32 = 3;
log::info!("count = {}", count);

let name = "ESP32-C6";
log::info!("ボードは {} です", name);
```

## コードを一行ずつ読む

- `let width = 40;` — 型注釈なしの宣言。コンパイラは整数と推論します（何も手がかりがない整数は `i32` という型になります）
- `let area = width * height;` — 右辺には計算式も書けます。計算した結果の 1000 が `area` に入ります
- `log::info!("面積は {} です", area);` — `{}` の場所に `area` の値が埋め込まれて表示されます。`{}` は「ここに値を入れる」という穴です
- `let count: u32 = 3;` — 型注釈つきの宣言。「`u32` にしたい」と人間が指定しています
- `let name = "ESP32-C6";` — 文字列も変数に入れられます（文字列の型の話は少し難しいので、この部では「表示に使える」とだけ覚えれば十分です）

## 実行方法

方法①なら、blinkyの `main` に貼って書き込みます。

```bash
cargo run
```

ログにこう表示されれば成功です（そのあとLチカのログが続きます）。

```text
INFO - 面積は 1000 です
INFO - count = 3
INFO - ボードは ESP32-C6 です
```

方法②なら、`log::info!` を `println!` に置き換えてPlaygroundで実行します。

## よくある失敗

**失敗1: 変数名を打ち間違えた（E0425）**

```rust
let width = 40;
log::info!("{}", widht); // widthのつもりがwidht
```

```text
error[E0425]: cannot find value `widht` in this scope
 --> src/main.rs:3:20
  |
3 |     println!("{}", widht);
  |                    ^^^^^
  |
help: a local variable with a similar name exists
  |
3 -     println!("{}", widht);
3 +     println!("{}", width);
```

「`widht` という値はこの範囲（scope）に見つからない」という意味です。注目してほしいのは `help:` の行です。Rustのコンパイラは間違いを指摘するだけでなく、「よく似た名前の変数があるよ。こう直せば?」と修正案まで出してくれます。エラーは敵ではなく、いちばん身近な先生です。**まず `error` の1行目を読み、次に `help` を探す**。この読み方をこの部全体で練習していきます。

**失敗2: 値をあとから変えようとした（E0384）**

```rust
let count = 0;
count = count + 1; // あとから変えようとする
```

```text
error[E0384]: cannot assign twice to immutable variable `count`
  |
2 |     let count = 0;
  |         ----- first assignment to `count`
3 |     count = count + 1;
  |     ^^^^^^^^^^^^^^^^^ cannot assign twice to immutable variable
  |
help: consider making this binding mutable
  |
2 |     let mut count = 0;
  |         +++
```

「変更不可（immutable）な変数 `count` に2回代入はできない」と言っています。実はRustの変数は、**最初から書き換え禁止**なのです。`help` が示すとおり `let mut` と書けば変更できるようになります。なぜそういう仕組みなのかは、まさに[次のページ](/embassy-esp32-c6/part02/02-mut/)のテーマです。

## やってみよう

面積の例に `depth = 10` という変数を追加して、`width * height * depth` で体積を計算し、`log::info!("体積は {} です", volume);` で表示してみましょう。1000 × 10 で 10000 と表示されれば成功です。

## 確認問題

1. `let x = 5;` の `x` の型は「ない」と言ってよいでしょうか?
2. 型注釈が必要になるのはどんなときでしょうか? 1つ挙げてください。
3. `log::info!("{}", speed);` で `speed` を宣言し忘れていたら、どのエラー番号が出るでしょうか?

<details>
<summary>答え</summary>

1. よくありません。型注釈を省略しただけで、コンパイル時には型推論によって必ず1つの型（この場合は `i32`）に決まっています。
2. コンパイラが右辺や使われ方から型を1つに決められないとき（ほかに、読む人に型を明示したいときにも書きます）。
3. E0425（cannot find value ... in this scope）。

</details>

## まとめ

- `let 名前 = 値;` で変数を宣言する。値の記憶場所はコンパイル時に型で決まる
- 型注釈 `: u32` は省略でき、コンパイラが型推論で決める。「型がない」のではない
- コンパイラエラーは `error` の1行目と `help` を読む。修正案まで教えてくれる
