## このページでできるようになること

- 固定長配列 `[i32; 5]` を作り、添え字で読み書きできる
- タプル `(u8, f32)` で型の違う値をまとめられる
- 範囲外アクセスに対するRustの動作（panic）を説明できる

## 先に結論

**配列**は「同じ型の値を決まった個数」並べる入れ物、**タプル**は「型が違ってもよい値をいくつか」束ねる入れ物です。配列の長さはコンパイル時に固定で、あとから伸ばせません。実はこれが組み込み向きの性質です。必要なメモリ量が最初から確定しているので、RAMがわずか512KBのESP32-C6でも安心して使えます。パソコン用Rustでよく使う伸び縮みする入れ物（VecやString）は、この教材の `no_std` 環境ではそのまま使えないため登場しません（事情は第5部で説明します）。配列とタプルだけで、この先の教材は十分戦えます。

## 身近なたとえ

配列は「卵のパック」です。10個用と決まったパックには卵が10個ちょうど。同じ形のものが決まった数だけ、番号つきの場所に並びます。タプルは「お弁当箱」です。ごはん・おかず・デザートと**種類の違うもの**をひとつの箱にまとめますが、どの仕切りに何が入るかは決まっています。

ただし実際の配列・タプルは、パックや弁当箱と違って中身の入れ替えに `mut` が必要で、場所の数え方が**0番から**始まります。

## 仕組み

**配列**の型は `[要素の型; 個数]` と書きます。個数まで型の一部です。

```rust
let temps: [i32; 5] = [21, 23, 25, 24, 22]; // 型注釈は省略可
let first = temps[0];   // 添え字は0から。これは21
let count = temps.len(); // 要素数5
let zeros = [0; 4];      // [0, 0, 0, 0] の省略記法
```

- 添え字（インデックス）は**0番から**。最後の要素は `temps[4]` です
- `[0; 4]` は「0を4個」。センサ読み値のバッファなどの初期化によく使います
- 範囲外の添え字を読むとどうなるかは「よくある失敗」で見ます

**タプル**は `( )` で値を並べます。

```rust
let sensor: (u8, f32) = (1, 25.5); // チャンネル番号と温度
let ch = sensor.0;    // 1つ目は .0
let value = sensor.1; // 2つ目は .1
let (ch, value) = sensor; // 分解して2つの変数へ（分配束縛）
```

「センサ番号と測定値」「ピン番号と状態」のように、**型は違うが常にセットで動く値**に向いています。関数から2つの値を返したいときにも使えます（第3部では、タプルに名前をつけて整理する `struct` に発展します）。

## Arduinoではどう書くか

C++の配列と見た目は似ています。

```cpp
int temps[5] = {21, 23, 25, 24, 22};
Serial.println(temps[10]); // 範囲外でもコンパイルも実行も通ってしまう!
```

決定的な違いは範囲外アクセスです。C++では `temps[10]` が**そのまま動いてしまい**、隣のメモリを読んだ謎の値が返ります。原因不明の誤動作の代表的な原因です。Rustは範囲外を検出して安全に停止します。なお、C++にタプルに当たる標準の手軽な書き方はなく、構造体を定義するのが普通でした。

## RustとEmbassyではどう書くか

これは抜粋です。貼りつけ先の完全なコードは examples/01-blinky を見てください。

```rust
let temps = [21, 23, 25, 24, 22];
log::info!("最初の温度: {}", temps[0]);
log::info!("3番目の温度: {}", temps[2]);
log::info!("個数: {}", temps.len());

let mut readings = [0; 4];
readings[0] = 512;
readings[1] = 518;
log::info!("readings[1] = {}", readings[1]);

let pin_state = (10, true);
log::info!("GPIO{} の状態は {}", pin_state.0, pin_state.1);

let (pin, is_on) = pin_state;
log::info!("ピン番号 {} / 点灯 {}", pin, is_on);

let sensor: (u8, f32) = (1, 25.5);
log::info!("センサ{} の値は {}", sensor.0, sensor.1);
```

## コードを一行ずつ読む

- `temps[2]` — 0から数えて2番、つまり**3番目**の要素（25）です
- `let mut readings = [0; 4];` — 書き換える配列には `mut` が必要です。0で埋めた4要素を用意し、測定のたびに `readings[i] = 値;` で上書きする、という使い方は第7部のADC（アナログ入力）でそのまま登場します
- `let pin_state = (10, true);` — 「GPIO10は点灯中」を1つの値に。型は `(i32, bool)` と推論されます
- `let (pin, is_on) = pin_state;` — タプルを一度に2つの変数へ分解します（**分配束縛**）。`.0` `.1` より名前がついて読みやすくなります

## よくある失敗

**失敗1: 範囲外の添え字を使った**

添え字が定数なら、コンパイラが先回りして止めてくれます。

```rust
let temps = [21, 23, 25, 24, 22];
log::info!("{}", temps[5]); // 最後は[4]なのに
```

```text
error: this operation will panic at runtime
  |
3 |     println!("{}", temps[5]);
  |                    ^^^^^^^^ index out of bounds: the length is 5 but the index is 5
```

「長さ5に対して添え字5は範囲外」。5個の配列の最後は `[4]` です（0から数えるため）。添え字が変数で、実行してみるまで分からない場合は、実行時にpanicで安全に停止します。

```text
panicked at src/main.rs:4:20:
index out of bounds: the len is 5 but the index is 10
```

C++のように黙って隣のメモリを読むのではなく、「範囲外に触れた」その場で分かるのがRustの動作です。組み込みでpanicが起きたときにどうなるか・どう付き合うかは第5部で扱います。

**失敗2: 宣言した個数と中身の個数が合わない（E0308）**

```rust
let temps: [i32; 5] = [21, 23, 25]; // 5個と言ったのに3個
```

```text
error[E0308]: mismatched types
  |
2 |     let temps: [i32; 5] = [21, 23, 25];
  |                --------   ^^^^^^^^^^^^ expected an array with a size of 5, found one with a size of 3
```

長さは型の一部なので、`[i32; 5]` と `[i32; 3]` は**別の型**です。だからいつもの型エラー（E0308）として捕まります。

**失敗3: タプルにない位置へアクセスした（E0609）**

```rust
let pin_state = (10, true);
log::info!("{}", pin_state.2); // 要素は.0と.1だけ
```

```text
error[E0609]: no field `2` on type `({integer}, bool)`
  |
3 |     println!("{}", pin_state.2);
  |                              ^ unknown field
  |
  = note: available fields are: `0`, `1`
```

タプルは要素数もコンパイル時に決まっているので、存在しない `.2` はその場でエラーです。`note` が「あるのは `0` と `1`」と教えてくれています。

## 確認問題

1. `[u8; 8]` と `[u8; 16]` は同じ型でしょうか?
2. 配列とタプルの使い分けを1文で説明してください。
3. C++と比べて、Rustの範囲外アクセスの動作はどう違い、なぜそのほうが安全なのでしょうか?

<details>
<summary>答え</summary>

1. 違う型です。配列は長さも型の一部です。
2. 同じ型の値が決まった個数並ぶなら配列、型の違う値をセットで持つならタプル。
3. C++は範囲外でも黙って隣のメモリを読み謎の値が返るが、Rustはコンパイル時に検出するか、実行時にpanicで即座に停止する。誤ったデータのまま動き続けるより、その場で気づける方が原因究明も安全性も上だからです。

</details>

## まとめ

- 配列 `[型; 個数]` は同じ型×固定個数。添え字は0から、長さは型の一部
- タプル `(A, B)` は型の違う値の束。`.0` / `.1` か分配束縛で取り出す
- 範囲外アクセスは隠れず止まる（コンパイルエラーまたはpanic）。これがRustの安全性の入り口
