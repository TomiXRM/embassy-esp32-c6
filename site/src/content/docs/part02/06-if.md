## このページでできるようになること

- `if` / `else if` / `else` で処理を分岐できる
- `let x = if 条件 { 値A } else { 値B };` のように、`if` を**式**として使える
- 条件が `bool` でないときのエラーを読める

## 先に結論

`if 条件 { ... } else { ... }` で、条件が `true` のときと `false` のときで処理を分けられます。条件は必ず `bool` で、C++と違って数値は使えません。条件のカッコ `( )` は不要ですが、中身の波カッコ `{ }` は1行でも省略できません。そしてRustの `if` は値を返す**式**なので、「条件によって変数に入れる値を変える」を1行で書けます。

## 身近なたとえ

`if` は「駅の改札」のような分岐点です。ICカードの残高が足りていればゲートが開いて直進、足りなければ横のチャージ機へ。どちらに進んでも、その先でまた合流して電車に乗ります。

ただし実際の `if` 式が改札と違うのは、「分岐した先から値を持って帰ってこられる」ことです。どちらの道を通ったかによって違う値が手元に残ります。

## 仕組み

基本の形は3段階です。

```rust
if 条件A {
    // Aがtrueのとき
} else if 条件B {
    // Aがfalseで、Bがtrueのとき
} else {
    // どちらもfalseのとき
}
```

条件は上から順に調べられ、最初に `true` になったブロックだけが実行されます。`else if` は何個でも並べられ、`else` は省略できます。

そして `if` は式なので、全体が値を持てます。

```rust
let wait_ms = if temp > 30 { 200 } else { 1000 };
```

「暑ければ200、そうでなければ1000を `wait_ms` に入れる」が1行で書けます。C++の三項演算子 `temp > 30 ? 200 : 1000` に相当しますが、Rustでは普通の `if` がそのままこの役割を兼ねます。値として使う場合は、**どの道を通っても同じ型の値になること**と、**`else` を省略しないこと**が条件です（理由は「よくある失敗」で見ます）。

## Arduinoではどう書くか

```cpp
if (temp > 30) {          // C++: 条件にカッコが必須
  Serial.println("hot");
}
int wait_ms = temp > 30 ? 200 : 1000;  // 値を選ぶには三項演算子
```

Rustでは条件のカッコを書かず、値の選択も同じ `if` で書けます。また、C++では `if (temp = 30)`（`==` のつもりが代入）が**コンパイルできてしまい**、有名なバグの温床でした。Rustでは型が合わずエラーになるので、この事故は起きません。

## RustとEmbassyではどう書くか

これは抜粋です。貼りつけ先の完全なコードは examples/01-blinky を見てください。

```rust
let temp = 28;
if temp > 30 {
    log::info!("暑すぎます");
} else if temp >= 20 {
    log::info!("快適です");
} else {
    log::info!("寒いです");
}

let wait_ms = if temp > 30 { 200 } else { 1000 };
log::info!("待ち時間は {} ミリ秒です", wait_ms);
```

## コードを一行ずつ読む

- `if temp > 30 {` — 条件は `bool` を返す式。カッコは不要、`{` は必須です
- `} else if temp >= 20 {` — 上の条件が `false` だったときだけ調べられます。`temp` が28なら、ここで `true` になり「快適です」が表示されます
- `let wait_ms = if ... { 200 } else { 1000 };` — `if` 式の結果を変数に入れています。ブロックの最後の式（`200` や `1000`。セミコロンなし）がブロックの値になる、という規則は関数のときと同じです。文全体の終わりには `;` が付きます

`wait_ms` のような値は、そのままblinkyの点滅間隔に使えます。`Timer::after(Duration::from_millis(wait_ms)).await` とすれば「暑いときは速く点滅」が実現できます。

## 実行方法

動かし方は2通りです（詳しくは[1. 変数とlet](/embassy-esp32-c6/part02/01-variables/)）。

```text
INFO - 快適です
INFO - 待ち時間は 1000 ミリ秒です
```

`temp` の値を 35 や 10 に変えて、表示が変わることを確かめてください。

## よくある失敗

**失敗1: 条件に数値を書いた（E0308）**

```rust
let temp = 28;
if temp { // 「tempが0でなければ」のつもり
    log::info!("暑い");
}
```

```text
error[E0308]: mismatched types
  |
3 |     if temp {
  |        ^^^^ expected `bool`, found integer
```

C++の「0以外はtrue」はRustにはありません。「何と比べたいのか」を `if temp > 0` のように必ず書きます。おなじみの `expected bool, found integer`（boolを期待したが整数が来た）です。

**失敗2: 値を返すifなのにelseがない（E0317）**

```rust
let temp = 28;
let wait_ms = if temp > 30 { 200 }; // elseがない
```

```text
error[E0317]: `if` may be missing an `else` clause
  |
3 |     let wait_ms = if temp > 30 { 200 };
  |                   ^^^^^^^^^^^^^^^---^^
  |                   |              |
  |                   |              found here
  |                   expected integer, found `()`
  |
  = note: `if` expressions without `else` evaluate to `()`
  = help: consider adding an `else` block that evaluates to the expected type
```

条件が `false` だったとき、`wait_ms` に入れる値がありません。Rustは「値が決まらない変数」を許さないので、`else` を要求します。`note` の「elseのないif式は `()` になる」が理由の説明、`help` の「同じ型を返すelseブロックを足して」が直し方です。両方の道で同じ型の値を返すようにしましょう。

## やってみよう

`let count = 7;` に対して、「3の倍数なら『3の倍数です』、そうでなければ数をそのまま表示」という分岐を書いてみましょう。3の倍数かどうかは `count % 3 == 0`（`%` は割り算の余り）で判定できます。

## 確認問題

1. Rustの `if` の条件に使える型は何でしょうか?
2. `let x = if flag { 1 } else { 2.5 };` はエラーになります。なぜでしょうか?
3. C++で起きる `if (a = 5)`（代入との取り違え）バグが、Rustで起きないのはなぜでしょうか?

<details>
<summary>答え</summary>

1. `bool` だけです。数値や他の型は使えません。
2. 2つの道が違う型（整数と小数）の値を返しているから。`if` を式として使うときは全ブロックが同じ型を返す必要があります。
3. 代入式 `a = 5` の値は `()` であり `bool` ではないため、E0308でコンパイルが止まるからです。

</details>

## まとめ

- `if 条件 { } else if { } else { }`。条件は `bool` のみ、`{ }` は必須
- `if` は式。`let x = if 条件 { A } else { B };` で値を選べる（両方同じ型、else必須）
- E0317は「falseのときの値がない」の意味。elseを足して直す
