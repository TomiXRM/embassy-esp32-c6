## このページでできるようになること

- `for i in 0..10` で決まった回数の繰り返しを書ける
- `0..5`（終端を含まない）と `1..=5`（含む）を使い分けられる
- 配列の要素を順に処理する `for` を書ける

## 先に結論

`for 変数 in 並び { ... }` は、「並び」から要素を1つずつ取り出して繰り返します。`0..5` は「0, 1, 2, 3, 4」を表す**range（レンジ、範囲）**で、`..` は終端を**含みません**。含めたいときは `0..=5` と書きます。カウンター変数を自分で用意して増やして比較する……という `while` での手作業が全部なくなるので、回数や要素数が決まっている繰り返しは `for` で書くのが基本です。

## 身近なたとえ

`for` は「出席簿の名前を上から順に読み上げる」繰り返しです。1人読んだら次の行へ。名簿が尽きたら自動で終わりです。「今何番目か」を自分で数えなくても、名簿が順番を管理してくれます。

ただし実際の `for` が読み上げと違うのは、名簿（rangeや配列）そのものを作るのもプログラムだという点です。`0..5` は「0から4までの名簿を作る」式です。

## 仕組み

```rust
for i in 0..5 {
    // iは0, 1, 2, 3, 4と変わりながら5回実行される
}
```

- `0..5` — 0以上5**未満**のrange。5回回りますが、`i` が5になることはありません
- `1..=5` — 1以上5**以下**のrange。「1回目〜5回目」と数えたいときに便利です
- `(0..5).rev()` — 逆順（4, 3, 2, 1, 0）。カウントダウンに使えます

「なぜ終端を含まないのか?」と思うかもしれません。プログラミングでは0から数え始めることが多く、「要素5個なら `0..5`」と個数がそのまま書けて、ずれの計算がいらないからです。

`for` のもうひとつの主戦場が配列（複数の値の並び。詳しくは次のページ）です。

```rust
let temps = [21, 23, 25, 24, 22];
for t in temps {
    // tに21, 23, 25, 24, 22が順に入る
}
```

`while` と違って「何番目まで進んだか」の管理をコンパイラに任せられるため、数え間違いによるバグ（1回多い/少ない、範囲外を読む）が起きません。

## Arduinoではどう書くか

C++の `for` は3つの部品を自分で書きました。

```cpp
for (int i = 0; i < 5; i++) {  // 初期化; 条件; 更新
  Serial.println(i);
}
```

Rustの `for i in 0..5` はこの3点セット（`i = 0` から始める、`i < 5` の間、`i++` で進める）を1つのrangeにまとめた形です。書く量が減るだけでなく、「条件を `<=` にしてしまい1回多く回る」「更新を書き忘れる」といった定番ミスの入り込む場所自体がなくなっています。

## RustとEmbassyではどう書くか

これは抜粋です。貼りつけ先の完全なコードは examples/01-blinky を見てください。

```rust
for i in 0..5 {
    log::info!("i = {}", i);
}

for i in 1..=3 {
    log::info!("{} 回目の点滅", i);
}

let temps = [21, 23, 25, 24, 22];
for t in temps {
    log::info!("温度: {}", t);
}

let mut total = 0;
for t in temps {
    total += t;
}
let average = total / temps.len() as i32;
log::info!("平均: {}", average);

for i in (0..5).rev() {
    log::info!("逆から: {}", i);
}
```

## コードを一行ずつ読む

- `for i in 0..5 {` — `i` は `let` なしで自動的に宣言され、ループの中だけで使えます
- `for i in 1..=3 {` — 「1回目」「2回目」「3回目」。人間の数え方に合わせたいときは `..=` が読みやすくなります
- `for t in temps {` — 配列から要素を順に取り出します。添え字（何番目か）は登場しません
- `total += t;` — 全要素の合計。`for` の外で `let mut total = 0;` を宣言しておくのがポイントです
- `temps.len() as i32` — `len()` は要素数(5)を返します。返る型が `usize`（添え字用の整数型）なので、`i32` の `total` と割り算するために `as` で変換しています

## よくある失敗

**失敗1: `0..5` に5が含まれると思い込む**

コンパイルエラーにならない失敗です。「`i = 5` まで表示されるはず」と思って実行すると、4で終わって戸惑います。`..` は終端を含まない、含めたければ `..=`。迷ったら上の実行結果のように、まずrangeだけをログに出して確かめるのが確実です。

**失敗2: 逆順のつもりで `5..0` と書く**

```rust
for i in 5..0 { // 1周も実行されない
    log::info!("{}", i);
}
```

これもエラーにならず、**何も表示されません**。rangeは「小さい方から大きい方へ」しか進めないので、`5..0` は空っぽの並びです。逆順は `(0..5).rev()` と書きます。「エラーが出ないのに動かない」ときは、ループが0回実行の可能性を疑ってください。

**失敗3: ループ変数を書き換えようとした（E0384）**

```rust
for i in 0..5 {
    i += 1; // 2つ飛ばしのつもり
}
```

```text
error[E0384]: cannot assign twice to immutable variable `i`
  |
2 |     for i in 0..5 {
  |         - first assignment to `i`
3 |         i += 1;
  |         ^^^^^^ cannot assign twice to immutable variable
```

`for` のループ変数も既定で変更不可です（E0384は[2. mutと変更できる変数](/embassy-esp32-c6/part02/02-mut/)で見ましたね）。そもそも `i` を手で進める必要はなく、2つ飛ばしなら `(0..10).step_by(2)` のようにrange側で表現します。

## やってみよう

九九の7の段を `for` で表示してみましょう。`for i in 1..=9` の中で `log::info!("7 x {} = {}", i, 7 * i);` とすれば、`7 x 1 = 7` から `7 x 9 = 63` まで並びます。

## 確認問題

1. `0..3` と `0..=3` はそれぞれ何回ループするでしょうか?
2. `for` が `while` より数え間違いに強いのはなぜでしょうか?
3. 配列 `temps` の全要素を表示する `for` 文を書いてください。

<details>
<summary>答え</summary>

1. `0..3` は3回（0, 1, 2）、`0..=3` は4回（0, 1, 2, 3）。
2. カウンターの初期化・条件・更新を自分で書かず、rangeや配列が「どこまで進んだか」を管理してくれるから。更新忘れや条件のずれが起きません。
3. `for t in temps { log::info!("{}", t); }`

</details>

## まとめ

- `for 変数 in 並び { }`。回数・要素数が決まった繰り返しの第一候補
- `..` は終端を含まない、`..=` は含む。逆順は `.rev()`
- 「エラーなしで動かない」ときは0回ループ（`5..0` など）を疑う
