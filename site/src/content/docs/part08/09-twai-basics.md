- TWAIとCANの名称の関係を説明できる
- 「コントローラ」と「トランシーバ」の役割の違いと、**トランシーバが必須**である理由を説明できる

## 先に結論

TWAI（Two-Wire Automotive Interface）は、Espressifが自社ペリフェラルに付けた名前で、**中身は自動車で広く使われるCAN（Controller Area Network、ISO 11898-1 / CAN 2.0）と互換**のプロトコルです。ESP32-C6はTWAIコントローラを2つ内蔵しますが、これは「フレームの組み立てと解釈」をする頭脳だけです。バスの2本線（CAN_H/CAN_L）に流れる**差動信号**を作る「口と耳」＝**トランシーバは内蔵されておらず、外付けが必須**です（公式ドキュメント明記）。**C6のGPIOをCAN_H/CAN_Lへ直結してはいけません**。バスの両端には120Ωの終端抵抗を付け、全ノードが同じビットレート（教材では500kbps）で動きます。

## 身近なたとえ

CANバスは「教室の全員が同じ1本の糸電話回線につながった連絡網」です。誰かが発言（フレーム送信）すると全員に届き、聞こえた人は「聞こえたよ」と即座に相槌（ACK）を打ちます。発言には話者名ではなく**話題番号（ID）**が付いていて、番号が小さい話題ほど優先されます。

ただし実際のCANでは、「声」は2本の電線の**電圧差**（差動信号）で運ばれ、声を電圧差に変換する専用の喉（トランシーバ）がないと回線に参加すらできない、という点がたとえとの違いです。

## 仕組み

### 名前の整理: CANとTWAI

| 名前 | 何を指すか |
|---|---|
| CAN | Bosch社が開発し自動車業界で標準化された通信規格（ISO 11898） |
| TWAI | EspressifがESP32系チップのCAN互換コントローラに付けた名称 |

商標上の理由でTWAIという別名が使われていますが、規格としてはCAN 2.0互換です。世の中の解説はCANという名前で書かれているので、「TWAIの資料が少ない」と思ったらCANで調べれば同じ知識が使えます。この教材ではC6の機能名としてTWAIを使います。

### コントローラとトランシーバ — 役割が違う2つの部品

```mermaid
graph LR
  subgraph ESP32-C6
    C[TWAIコントローラ<br>フレームの組み立て・ACK・エラー管理]
  end
  subgraph 外付け部品（必須）
    T[CAN/TWAIトランシーバ<br>TJA1051等 3.3V品<br>論理信号⇔差動信号の変換]
  end
  C -->|TX 論理信号| T
  T -->|RX 論理信号| C
  T ---|CAN_H| BUS[バス2本線]
  T ---|CAN_L| BUS
```

- **TWAIコントローラ**（C6に2つ内蔵）: フレームのビット列を組み立て、送信の調停、ACKの生成、エラーの数え上げまでを行うデジタル回路です。入出力は普通の3.3V論理信号（TX/RX）です
- **CAN/TWAIトランシーバ**（外付け必須）: TX/RXの論理信号と、バス上の**差動信号**（CAN_HとCAN_Lの電圧差）を相互変換するアナログ部品です。公式ドキュメントは「ESP32-C6は内蔵トランシーバを持たないため外付けが必須」と明記し、TJA105x系（ISO 11898-2準拠）を例に挙げています。C6は3.3Vロジックなので**3.3V対応品（TJA1051/3など）**を選びます

**C6のピンをCAN_H/CAN_Lへ直結することは絶対にしてはいけません。** バスの差動電圧や他ノードの駆動はGPIOの定格を超えることがあり、チップを壊す恐れがあります。TXとRXは必ずトランシーバのTXD/RXDにつなぎます。

### バスの形と終端抵抗120Ω

```text
        120Ω                                     120Ω
   ┌────[抵抗]────┬──────────────┬──────────[抵抗]────┐
   │   CAN_H      │              │              CAN_H │
   ●──────────────●──────────────●────────────────────●
   ●──────────────●──────────────●────────────────────●
   │   CAN_L      │              │              CAN_L │
   └──────────────┴──────────────┴────────────────────┘
     ノード1        ノード2         ノード3
  （各ノード = C6 + トランシーバ の組）
```

- 全ノードが同じ2本線（CAN_H/CAN_L）に並列にぶら下がる**バス型**の配線です
- **バスの両端に120Ωの終端抵抗**を1本ずつ（合計2本）付けます。信号の反射を抑えるためで、これがないと通信が不安定になります。市販のトランシーバモジュールには終端抵抗を実装済みのものもあるので、重複しないよう確認します

### ビットレート・フレーム・ACK

- **ビットレート**: 全ノード共通の速度です。125k/250k/500k/1Mbpsなどが定番で、教材では500kbps（`BaudRate::B500K`）を使います。UARTと同じく、食い違うと通信できません
- **フレーム**: 1回の送信単位です。**ID（標準11ビット）＋データ最大8バイト**が中心で、IDは「宛先」ではなく「内容の種類」を表します。IDが小さいほど優先度が高く、同時送信の衝突はIDの比較で自動的に調停されます
- **ACK**: フレームを正しく受信した**どれか1つ以上のノード**が、フレーム末尾のACKスロットで信号を返します。ACKが1つも返らない送信は失敗として扱われます。1台だけでは送信が常に失敗するのはこのためです（次ページのセルフテストモードで回避します）

### エラー状態とバスオフ

CANは「壊れたノードがバス全体を道連れにしない」仕組みを持ちます。各コントローラはエラーを数えており、状態が3段階に変わります。

| 状態 | 意味 |
|---|---|
| エラーアクティブ | 正常。送受信に全面参加できる |
| エラーパッシブ | エラーが目立つ。エラー通知を控えめにし、送信も遠慮がちになる |
| **バスオフ** | エラー多発。**バスから切り離され、送信できなくなる** |

配線ミスやビットレート違いで送信失敗が積み重なると、コントローラは自らバスオフへ移行します。「急に何も送れなくなった」ときはこの状態を疑います。

## RustとEmbassyではどう書くか

用語が配線と結びついたところで、設定コードの形だけ先に見ておきます。これは抜粋です。完全なコードは `examples/11-twai` を見てください。

```rust
use esp_hal::twai::{BaudRate, TwaiConfiguration, TwaiMode};

// TWAI0を500kbps・セルフテストモードで設定する。
// new_no_transceiver()はトランシーバなしでピン同士を直結する構成用。
// 引数の順番は「RXピン, TXピン」なので注意！
let twai_config = TwaiConfiguration::new_no_transceiver(
    peripherals.TWAI0,
    peripherals.GPIO3, // RX
    peripherals.GPIO2, // TX
    BaudRate::B500K,
    TwaiMode::SelfTest,
)
.into_async();
```

`new_no_transceiver`という名前が示すとおり、これは**トランシーバなしの学習用構成**（TXとRXのピン直結）専用です。実バスに参加するときの構成は次ページで扱います。なおTWAIはesp-halの`unstable` feature配下のAPIで、将来のバージョンで変わる可能性があります。

## 実行方法

このページは用語と配線の理解が主役です。動かすのは次ページでまとめて行います。手元にトランシーバがなくても、次ページのセルフテストモードなら**C6一台とジャンパ線1本**で送受信を体験できます。

## よくある失敗

- **C6のTX/RXピンをCAN_H/CAN_Lへ直結する**: 最も危険な間違いです。差動バスの電圧はGPIOの定格を超えることがあり、チップを壊す恐れがあります。TX/RXは必ずトランシーバのTXD/RXDへ
- **5V専用トランシーバ（ロジックも5V）を使う**: TXD/RXDが5V振幅だとC6の絶対最大定格3.6Vを超えます。TJA1051/3のような3.3Vロジック対応品を選びます
- **終端抵抗を忘れる／3本以上付ける**: 反射でエラーフレームが多発し、最悪バスオフに至ります。120Ωは「バスの両端に2本だけ」です
- **ノード間でビットレートが違う**: 全フレームがエラーになり、エラーカウントが積み上がってバスオフになります

## やってみよう

手元のCAN搭載製品を1つ調べてみましょう。たとえば自動車のOBD-IIポート（診断コネクタ）にはCANの端子があります。「何kbpsか」「IDに何が割り当てられているか」を検索すると、今日の用語がそのまま出てきます（実車への接続はしないでください）。

## 確認問題

1. TWAIコントローラとトランシーバの役割の違いを一言ずつで説明してください。
2. 終端抵抗120Ωは、どこに何本必要ですか。
3. 1台のノードだけでフレームを普通に送信すると失敗します。なぜですか。

<details>
<summary>答え</summary>

1. コントローラはフレームの組み立て・調停・エラー管理をするデジタルな頭脳、トランシーバは論理信号とバスの差動信号を変換するアナログな口と耳です。C6は前者のみ内蔵し、後者は外付け必須です。
2. バスの両端に1本ずつ、合計2本です。
3. CANではフレームを受信した他ノードがACKを返す決まりで、1台きりだとACKを返す相手がいないため送信が失敗扱いになるからです。

</details>

## まとめ

- TWAIはEspressifの名称で、中身はCAN（ISO 11898-1）互換。世間の知識はCANの名前で調べられる
- C6はコントローラ×2のみ内蔵。**トランシーバ（3.3V品）は外付け必須**で、GPIOのCAN_H/CAN_L直結は厳禁
- バスは2本線＋両端120Ω終端。全ノード同一ビットレート、受信ノードがACKを返し、エラー多発ノードはバスオフで自主退場する
