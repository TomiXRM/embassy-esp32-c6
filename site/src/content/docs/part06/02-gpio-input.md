## このページでできるようになること

- 入力モードのGPIOピンで何が読めるかを説明できる
- `Input`型でピンを構え、`is_high()`/`is_low()`で状態を読める
- 何ボルト以上がHigh、何ボルト以下がLowと判定されるかが分かる

## 先に結論

入力モードのGPIOピンは、外から加えられた電圧を「High（1）かLow（0）か」の2値として読み取ります。esp-halでは`Input::new(ピン, InputConfig)`で入力ピンを作り、`is_high()`/`is_low()`で今の状態を読めます。ESP32-C6では、おおよそ電源電圧の75%（約2.5V）以上がHigh、25%（約0.8V）以下がLowと判定されます。どちらにもつながっていない「浮いた」ピンの値は当てにならない、という重要な注意があります（対策は次のページで学びます）。

## 身近なたとえ

出力が「スイッチを操作する側」だったのに対し、入力は「スイッチの状態を目で見る側」です。廊下の電気が点いているか消えているかを確認する係だと思ってください。係は照明を操作しません。見るだけです。

実際のマイコンでは、係が見ているのは照明ではなく**ピンの電圧**です。そして「点いている／消えている」の2択に無理やり分類します。中途半端な明るさ（中間の電圧）は、どちらに分類されるか保証されません。ここが本物の目視と違うところです。

## 仕組み

入力ピンは、ピンの電圧をチップ内の**入力レジスタ**に映します。プログラムが`is_high()`を呼ぶと、そのレジスタを読んで判定した結果が返ります。

```mermaid
graph LR
  A[外部回路<br>ボタン・センサなど] --> B[物理ピン]
  B --> C[入力バッファ]
  C --> D[GPIO入力レジスタ]
  D --> E[プログラム<br>is_high / is_low]
```

「何ボルトならHighか」はデータシートで決まっています。

| 判定 | 条件（VDD=3.3Vのとき） |
|---|---|
| High（1）と判定 | 0.75×VDD以上 ＝ 約2.5V以上 |
| Low（0）と判定 | 0.25×VDD以下 ＝ 約0.8V以下 |
| その中間 | **保証されない**（HighともLowとも読めうる） |

中間の電圧が入り続ける配線は避ける、というのが入力回路の基本です。特に「どこにもつながっていないピン」は、周囲のノイズを拾って中間の電圧をふらふらと動きます。これを**浮いた（フローティング）入力**と呼び、次のページの主役になります。

## RustとEmbassyではどう書くか

入力ピンを作って読む部分だけを見ます。これは抜粋です。完全なコードは `examples/02-button` を見てください。

```rust
use esp_hal::gpio::{Input, InputConfig, Pull};

// BOOTボタン（GPIO9）を入力に設定。
// 内部プルアップの意味は次のページで説明します。
let config = InputConfig::default().with_pull(Pull::Up);
let mut button = Input::new(peripherals.GPIO9, config);

// 今この瞬間の状態を読む
if button.is_low() {
    info!("ボタンは押されています");
}
```

## コードを一行ずつ読む

```rust
let config = InputConfig::default().with_pull(Pull::Up);
```

入力の設定です。`with_pull(Pull::Up)`は「内部プルアップ抵抗を有効にする」という指定で、浮いた入力を防ぐ仕掛けです。詳しくは次のページで説明するので、ここでは「入力にはこの指定がほぼ必須」とだけ覚えてください。

```rust
let mut button = Input::new(peripherals.GPIO9, config);
```

`Output::new`と同じ形で、ピンの所有権を渡して`Input`型を作ります。GPIO9はボード上のBOOTボタンにつながっているピンです。

```rust
button.is_low()
```

ピンが今Lowなら`true`を返します。対になる`is_high()`もあります。BOOTボタンは「押すとピンがGNDへつながる」配線なので、**押されている＝Low**です。直感と逆に感じるかもしれませんが、理由は次のページとボタンのページで分かります。

## よくある失敗

- **`Input`にしたピンで`set_high()`を呼ぼうとしてコンパイルエラー**: `Input`型に出力用メソッドはありません。読む係と操作する係は型で分かれています。逆に言えば「入力のつもりのピンをうっかり駆動する」バグが型で防がれています
- **何もつないでいないピンを読んだら値がでたらめに変わる**: 浮いた入力です。故障ではなく、どこにもつながっていないピンの宿命です。プルアップ／プルダウン（次ページ）で解決します
- **3.3Vを超える電圧を入力してしまう**: ESP32-C6の入力の絶対最大は3.6Vです。5Vのセンサ出力などを直結するとチップを壊す恐れがあります。つなぐ前に相手の出力電圧を必ず確認してください

## やってみよう

`examples/02-button`を書き込み、BOOTボタンを押しながら動作を観察してください。そのあとコードの`is_low()`を`is_high()`に変えると、押しても反応しなくなります。「押す＝Low」を体で確認する実験です（確認したら元に戻しましょう）。

## 確認問題

1. VDD=3.3Vのとき、1.5Vの電圧が入力ピンに加わりました。HighとLowのどちらに読まれますか。
2. `Input`型のピンに`set_high()`が存在しないことは、どんなバグを防いでいますか。

<details>
<summary>答え</summary>

1. どちらに読まれるか**保証されません**。High判定は約2.5V以上、Low判定は約0.8V以下で、1.5Vはその中間だからです。こうした中間電圧が入り続ける設計は避けます。
2. 入力として使うつもりのピンを、コードの誤りで出力として駆動してしまうバグです。センサやボタンがつながったピンを駆動すると回路の衝突（ショートに近い状態）が起きえますが、型の区別によりコンパイル時に防がれます。

</details>

## まとめ

- 入力ピンは電圧をHigh/Lowの2値として読む。`Input::new`で作り、`is_high()`/`is_low()`で読む
- High判定は約0.75×VDD以上、Low判定は約0.25×VDD以下。中間は保証されない
- どこにもつながっていない「浮いた」入力は値が当てにならない
