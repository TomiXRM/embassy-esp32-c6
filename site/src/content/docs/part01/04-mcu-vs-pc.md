## このページでできるようになること

- マイコンとパソコンの違いを「メモリ」「OS」「役割」の3点で説明できる
- フラッシュとRAM（SRAM）の役割の違いを説明できる
- 「OSがない」ことが開発に与える影響を挙げられる

## 先に結論

マイコンは「小さくて遅いパソコン」ではなく、**役割の違う機械**です。パソコンは人間のためにいろいろなアプリを切り替えて動かす汎用機、マイコンは決まった仕事を確実にこなし続ける専用機です。ESP32-C6のRAMは512KB（パソコンの数万分の1）、そして**OSがありません**。OSがないとは、あなたのプログラムがチップ上で動く**唯一の**プログラムであり、ファイルもウィンドウも「他のアプリ」も存在しないということです。この不便さは、裏返すと「すべてを自分が制御できる」という強みです。

## 身近なたとえ

パソコンは「何でも屋の大きなオフィス」です。managerであるOSがいて、たくさんの社員（アプリ）に机や時間を割り振ります。マイコンは「一人親方の作業小屋」です。managerはいません。小屋にいるのはあなたのプログラムだけで、道具（ペリフェラル）も電気も全部あなたが直接管理します。

たとえと違うのは、作業小屋の職人は疲れ知らずで、電源が入っている限り同じ仕事を何年でも続けることです。

## 仕組み — 数字で比べる

| 項目 | ふつうのパソコン | ESP32-C6 |
|---|---|---|
| RAM（作業メモリ） | 16GBなど | **HP SRAM 512KB**（約3万分の1） |
| プログラム置き場 | SSD 512GBなど | **フラッシュ 8MB**（ボード搭載モジュール） |
| CPU | 数GHz×多コア | RISC-V 最大160MHz |
| OS | Windows / macOS / Linux | **なし**（プログラムが直接動く） |
| 同時に動くプログラム | 何十個 | **あなたの1個だけ** |

### フラッシュとRAMの役割分担

- **フラッシュ（8MB）**: プログラム本体の置き場。電源を切っても消えません。書き込みツールで「書き込む」先はここです
- **RAM（SRAM 512KB）**: 実行中の変数などの作業場。電源を切ると消えます

パソコンの「SSDとメモリ」の関係とほぼ同じですが、量が桁違いに少ないため、「大きな配列をどんと確保する」「データを際限なくためる」といった書き方がすぐ限界に達します。Rustでこの制約とどう付き合うかは第5部で扱います。

### 「OSがない」の意味

OSがないと、次のものが（最初は）ありません。

- **ファイルシステム**: `open("data.txt")`のような操作は存在しません
- **プロセスとスケジューラ**: アプリを切り替えてくれる仕組みはありません。複数の仕事をこなす仕組みは自分で用意します — それがEmbassyです（第9部）
- **画面とキーボード**: 入出力はGPIOやシリアル通信で自分で行います
- **終了という概念**: プログラムに「終わり」がなく、無限ループで動き続けます。電源を切るまでが実行時間です

代わりに得られるのは、**起動の速さ**（OSの起動を待たない）、**動作の予測しやすさ**（他のアプリに割り込まれない）、そして**消費電力の低さ**です。電池で何か月も動く機器は、この性質の上に成り立っています。

## よくある失敗

- **パソコンの感覚でメモリを使う**: 「とりあえず1MBのバッファを確保」はC6ではRAMの2倍でありコンパイルすら通りません。マイコンでは使うメモリ量を設計時に決めるのが基本です
- **`println!`でファイルに保存しようとする**: ログの出力先はファイルではなくシリアル通信（USB経由でパソコンに表示）です。第9ページで実際に使います
- **プログラムが「終了しない」ことに戸惑う**: マイコンの`main`は戻りません。Lチカのコードで`-> !`という見慣れない戻り値が出てくるのはこのためです

## やってみよう

自分のパソコンのRAM容量を調べて、512KBの何倍かを計算してみてください（16GBなら約3万3千倍です）。この比率を体感しておくと、第5部のメモリの話がぐっと現実的になります。

## 確認問題

1. フラッシュとRAM（SRAM）の役割の違いを一言ずつで説明してください。
2. 「OSがない」環境に存在しないものを3つ挙げてください。
3. OSがないことの利点を1つ挙げてください。

<details>
<summary>答え</summary>

1. フラッシュはプログラムの置き場（電源を切っても消えない）、RAMは実行中の作業場（電源を切ると消える）です。
2. 例: ファイルシステム、プロセス切り替え（スケジューラ）、画面・キーボード、「プログラムの終了」という概念、他のアプリ。
3. 例: 起動が速い、動作が予測しやすい、消費電力を低くできる、すべてを自分で制御できる。

</details>

## まとめ

- マイコンは専用機。C6はRAM 512KB・フラッシュ8MB・OSなし
- フラッシュ＝プログラム置き場、RAM＝作業場。量はパソコンの数万分の1
- OSがない不便さは、速さ・予測可能性・省電力という強みの裏返し
