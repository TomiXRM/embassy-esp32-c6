## このページでできるようになること

- espflashでプログラムをボードへ書き込める
- シリアルモニタでマイコンからのログを読める
- ボードの2つのUSBポートの違いが分かる
- 書き込みに失敗したときにBOOTボタンで復旧できる

## 先に結論

書き込みは`espflash flash --monitor`で行いますが、前のページの設定のおかげで、実際に打つのは`cargo run --release`だけです。ビルド→書き込み→シリアルモニタ表示まで一気に進みます。ボードにはUSB-Cポートが2つあり、**どちらからでも書き込めます**。書き込みに失敗するときは、BOOTボタン（GPIO9）を押しながらリセットすると「書き込みモード」に入れます。

## 身近なたとえ

書き込み（フラッシュへの書き込み）は「マイコンの本棚に、新しい手順書を差し替える作業」です。マイコンは電源が入るたびに、本棚（フラッシュメモリ）の手順書を最初から読んで実行します。シリアルモニタは、マイコンが作業しながら書く「日報」をパソコン側でのぞく窓です。

たとえと違うのは、差し替え中は古い手順書が消され、途中で電源を抜くと中途半端な状態になることです。書き込み中はケーブルを抜かないでください。

## 仕組み — 書き込みの流れ

```mermaid
graph LR
  A["cargo build<br>（コンパイル）"] --> B["espflash flash<br>（USB経由で書き込み）"]
  B --> C["リセット後、<br>新プログラムが起動"]
  C --> D["espflash monitor<br>（ログを表示）"]
```

`cargo run`は、この一連の流れを自動で行います。`.cargo/config.toml`の`runner = "espflash flash --monitor --chip esp32c6"`という設定が効いているためです。

### 2つのUSBポート

ESP32-C6-DevKitC-1にはUSB-Cポートが2つあります。

| ポートの刻印 | 中身 | 特徴 |
|---|---|---|
| UART | CP2102Nチップ経由のシリアル変換（UART0 = GPIO16/17） | 昔ながらの確実な経路。Windowsはドライバが必要なことがある |
| USB | チップ内蔵のUSB Serial/JTAG（GPIO12/13） | 変換チップを通らない直結。ドライバ不要なことが多い |

どちらでも書き込み・モニタができます。迷ったら**USB側**でかまいません。

## 実行方法

プロジェクトのフォルダで実行します。

```bash
cargo run --release
```

ポートが複数見つかると、espflashがどれを使うか聞いてきます。矢印キーで選んでEnterを押してください（`usbserial`や`SLAB`を含む名前がUART側、`usbmodem`等がUSB側です。表示はOSにより異なります）。

書き込みの進行表示のあと、自動でモニタに切り替わり、プログラムのログが流れます。前のページで作ったプロジェクト（または教材の`examples/01-blinky`）なら、次のような表示が出ます。

```text
INFO - Lチカを開始します
```

モニタを終了するにはCtrl+Cを押します。マイコン側のプログラムは動き続けます（モニタは「日報をのぞく窓」なので、閉じても作業は止まりません）。

ログの量は環境変数`ESP_LOG`で変わります。`.cargo/config.toml`に`ESP_LOG = "info"`と設定済みなので、`info!`マクロで書いたログが表示されます。

## 書き込みモードとBOOTボタン

ふだんespflashは自動でボードを「書き込みモード」に切り替えます。まれに失敗するときは手動で入れます。

1. **BOOTボタン**（GPIO9につながったボタン）を押したままにする
2. そのまま**RSTボタン**を短く押して離す
3. BOOTボタンを離し、もう一度`cargo run --release`

これはGPIO9のレベルで起動モードが決まるというC6の仕組みを使った操作です。BOOTを押しながらリセットすると、プログラムを実行する代わりに書き込み待ち状態で起動します。

## よくある失敗

- **`Error: ... serialport ... `やポートが1つも見つからない**: ①ケーブルがデータ通信対応か、②Linuxならポートの権限（[7. 開発環境の構築](/embassy-esp32-c6/part01/07-setup/)参照）、③WindowsのUARTポートならCP210xドライバ、の順で確認してください
- **書き込みが途中で`Connection failed`になる**: 上のBOOTボタンの手順で書き込みモードに入れてから再実行してください。USBハブを介している場合は直挿しも試します
- **書き込みは成功したのにログが何も出ない**: モニタを開く前の出力は見えていないだけのことがあります。RSTボタンを押すと最初からログが流れます
- **2回目の`cargo run`で「ポートが開けない」エラー**: 前のモニタが開いたままです。前のターミナルでCtrl+Cを押して閉じてください

## やってみよう

書き込みが終わってモニタが表示されている状態で、ボードのRSTボタンを押してみてください。起動時のログが最初から流れ直すはずです。「リセット＝本棚の手順書を最初から読み直す」を体感できます。

## 確認問題

1. `cargo run --release`一発で書き込みとモニタ表示まで進むのは、どのファイルのどの設定のおかげですか。
2. BOOTボタンを押しながらリセットすると何が起きますか。
3. シリアルモニタを閉じると、マイコンのプログラムは止まりますか。

<details>
<summary>答え</summary>

1. `.cargo/config.toml`の`runner = "espflash flash --monitor --chip esp32c6"`という設定です。`cargo run`がビルド後にこのコマンドを実行します。
2. GPIO9がLowの状態で起動するため、プログラムを実行せず「書き込み待ちモード」で立ち上がります。書き込みが失敗するときの復旧手段です。
3. 止まりません。モニタはログを表示しているだけで、プログラムはマイコン上で独立して動き続けます。

</details>

## まとめ

- 書き込み＋モニタは`cargo run --release`一発（実体は`espflash flash --monitor`）
- USBポートは2つ。どちらでも書き込め、迷ったらUSB（Serial/JTAG）側
- 書き込みに失敗したら、BOOT（GPIO9）を押しながらRST→BOOTを離して再実行
